Entrega Semana 5

Primeira versão funcional para predição de churn

Esta entrega cobre a análise exploratória da base de clientes e uma primeira versão funcional do sistema: upload de CSV, treino do modelo no backend, API de inferência e interface web simples para simular o risco de cancelamento.

EDA Upload CSV RandomForest FastAPI Vercel + Render Inferência com mínimo de 2 campos
Arquitetura

Como o sistema funciona

Dataset Usuário envia gym_churn_us.csv pela aba Upload CSV.
Treino Backend valida colunas, limpa dados, treina RandomForest e calcula métricas.
Backend FastAPI no Render mantém o modelo treinado em runtime e expõe /predict.
Frontend Vercel envia dados para inferência e exibe probabilidade e risco.
Dataset

Base utilizada

  • 4.000 clientes de uma academia americana.
  • Target: Churn, em que 0 significa permaneceu e 1 significa cancelou.
  • Sem valores nulos, com limpeza de duplicados no pipeline.
  • gender e Phone foram descartadas por baixa correlação com churn.

Variáveis usadas pelo modelo

EDA

Análises previstas

  • Distribuição da variável Churn para entender o desbalanceamento.
  • Comparação de idade, tempo como cliente e frequência entre churn e não churn.
  • Análise de contrato e meses até vencimento como sinais de retenção.
  • Correlação das features com Churn para justificar seleção de variáveis.
Modelo

Primeira versão preditiva

Utilizamos o RandomForestClassifier, um algoritmo consolidado de machine learning disponível na biblioteca scikit-learn. Ele combina múltiplas árvores de decisão para estimar a probabilidade de churn de cada cliente.

Sua performance foi avaliada principalmente pelo ROC-AUC, métrica adequada para medir a capacidade de diferenciar clientes com maior ou menor risco de churn.

O treino acontece após o upload do CSV, com divisão estratificada de treino e teste. As métricas do treino atual aparecem automaticamente na aba Upload CSV após o backend finalizar o treinamento.
Inferência

Como interpretar o resultado

  • Baixo risco: probabilidade menor que 40%.
  • Risco médio: probabilidade entre 40% e 69%.
  • Alto risco: probabilidade a partir de 70%.
Antes de inferir, envie o CSV na aba Upload CSV. A tela permite calcular com pelo menos 2 campos; os campos não informados são preenchidos com valores de referência derivados do treinamento.
