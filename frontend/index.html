Entrega Semana 5

Primeira versão funcional para predição de churn

Esta entrega cobre a análise exploratória da base de clientes e uma primeira versão funcional do sistema: modelo preditivo, API de inferência e interface web simples para simular o risco de cancelamento.

EDA RandomForest FastAPI Vercel + Render Inferência com mínimo de 2 campos
Arquitetura

Como o sistema funciona

Dataset Base local em dataset/gym_churn_us.csv usada para análise e treino.
Treino train_model.py limpa os dados, treina o modelo e gera model.pkl.
Backend FastAPI no Render carrega model.pkl e expõe o endpoint /predict.
Frontend Vercel envia os dados do formulário e exibe probabilidade e risco.
Dataset

Base utilizada

  • 4.000 clientes de uma academia americana.
  • Target: Churn, em que 0 significa permaneceu e 1 significa cancelou.
  • Sem valores nulos, com limpeza de duplicados no pipeline.
  • gender e Phone foram descartadas por baixa correlação com churn.
Features

Variáveis usadas pelo modelo

Lifetime Avg_class_frequency_current_month Age Contract_period Month_to_end_contract Avg_class_frequency_total Avg_additional_charges_total Group_visits, Promo_friends, Partner, Near_Location
EDA

Análises previstas

  • Distribuição da variável Churn para entender o desbalanceamento.
  • Comparação de idade, tempo como cliente e frequência entre churn e não churn.
  • Análise de contrato e meses até vencimento como sinais de retenção.
  • Correlação das features com Churn para justificar seleção de variáveis.
Modelo

Primeira versão preditiva

O modelo inicial é um RandomForestClassifier treinado com divisão estratificada de treino e teste. A métrica principal para validação é ROC-AUC, acompanhada de precision, recall e classification report.

Quando o treino final for executado, inserir aqui o ROC-AUC e as principais métricas obtidas.
Inferência

Como interpretar o resultado

  • Baixo risco: probabilidade menor que 40%.
  • Risco médio: probabilidade entre 40% e 69%.
  • Alto risco: probabilidade a partir de 70%.
A tela permite calcular com pelo menos 2 campos. Os campos não informados são preenchidos com valores de referência, então a estimativa fica mais confiável quando mais dados reais são preenchidos.
Limitações

O que ainda será evoluído

  • Semana 10 deve trazer modelo final e explicabilidade com LLM.
  • Os valores de referência para campos ausentes são uma simplificação para a primeira versão.
  • O modelo ainda precisa ser validado com as métricas finais após o treino definitivo.
